# API 密钥管理

## 文件说明

### `.env.keys` ⚠️
- **包含所有 API 密钥的完整副本**
- **已添加到 `.gitignore`，绝对不能上传到 GitHub**
- 用于本地开发和部署时复制环境变量
- 此文件泄露 = 所有 API 密钥泄露，请妥善保管

### `.env.local`
- 本地开发使用的环境变量
- 从 `.env.keys` 复制而来
- 已添加到 `.gitignore`

### `.env.example`
- 环境变量模板（不含真实密钥）
- 可以安全上传到 GitHub
- 新成员首次 setup 时参考此文件

---

## 本地开发使用

```bash
# 1. 首次 setup 时，从 .env.keys 复制到 .env.local
cp .env.keys .env.local

# 2. 启动开发服务器
npm run dev
```

---

## 部署时使用

### Render (后端)
1. 打开 `.env.keys` 文件
2. 复制以下变量到 Render Dashboard → Environment：
   - `ANTHROPIC_API_KEY`
   - `ANTHROPIC_BASE_URL`
   - `TAVILY_API_KEY`
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `SUPABASE_SERVICE_ROLE_KEY`

### Vercel (前端)
1. 打开 `.env.keys` 文件
2. 复制以下变量到 Vercel Dashboard → Environment Variables：
   - `NEXT_PUBLIC_RENDER_API_URL`

## 安全提醒

### ❌ 绝对不要
- 将 `.env.keys` 上传到 GitHub
- 将 `.env.keys` 发送给任何人
- 在代码中硬编码 API 密钥
- 在 commit 信息中包含 API 密钥

### ✅ 应该做
- 只从 `.env.keys` 读取密钥
- 部署时使用环境变量
- 定期检查 `.gitignore` 确保 `.env.keys` 被忽略
- 如果密钥泄露，立即在对应平台重置

---

## 密钥重置

如果怀疑密钥泄露，立即在以下平台重置：

| 服务 | 重置链接 |
|------|----------|
| Supabase | https://app.supabase.com/project/_settings/api |
| Tavily | https://app.tavily.com/home |
| Alibaba DashScope | https://dashscope.console.aliyun.com/apiKey |
| Stripe | https://dashboard.stripe.com/apikeys |

重置后，更新 `.env.keys` 文件和所有部署平台的环境变量。
